# Архитектура сессий

## Session ID

Сессия идентифицируется `session_id` в `event-collector.py`:

1. **Приоритет 1:** `CLAUDE_SESSION_ID` из env (устанавливает Claude Code)
2. **Приоритет 2:** `session_id` из поля события
3. **Fallback:** `md5(cwd + "_" + VSCODE_PID)[:12]`
   - `VSCODE_PID` — PID окна VS Code, стабилен пока вкладка открыта
   - Детерминирован: тот же ввод → тот же ID при рестарте хука

> ⚠️ Не использовать `os.getppid()` — меняется при каждом рестарте хука, создаёт фантомные сессии

## Дедупликация (`api/state.js`)

Сессии с одинаковым `cwd + session_type` объединяются в одну если разница `started_at` < 300 секунд (TTL сессии). Это предотвращает дубликаты при reconnect хука.

## TTL

Сессия считается активной пока `last_event_ts` < 300 секунд назад. При отсутствии событий (Claude Code молчит) — исчезает сама.

## Типы сессий

| Тип | Признак | Отображение |
|-----|---------|-------------|
| `vscode` | `VSCODE_PID` в env | Бейдж VSCODE |
| `cli` | нет `VSCODE_PID` | Бейдж CLI |

Субагенты (Agent tool) наследуют `VSCODE_PID` родителя → отображаются как `vscode`. Это ожидаемое поведение.
